Покажи и кратко резюмируй действующие backend-правила проекта.

Сначала:
1. Прочитай `CLAUDE.md`, если он есть.
2. Прочитай все файлы в `.claude/rules/backend/`.
3. Если у rules есть `paths`, учитывай только те, что относятся к текущим файлам.

Затем:
- выдай краткий список активных правил;
- отдельно перечисли правила по стилю кода, тестам, API и БД;
- перед началом любых backend-изменений напомни эти правила.
